# Preguntas del jurado: AMARU KIDS

## 1. Que problema resuelve AMARU KIDS?

AMARU KIDS reduce barreras de aprendizaje digital para ninos con discapacidad visual, auditiva, dificultades de comunicacion, TDAH o necesidades educativas especificas, especialmente cuando las plataformas educativas no se adaptan a sus formas de aprender.

## 2. Cual es el usuario principal?

El usuario principal es un nino de educacion inicial o basica que necesita apoyos de accesibilidad. Los usuarios secundarios son familias, docentes, terapeutas, fundaciones y escuelas inclusivas.

## 3. Que hace diferente a AMARU KIDS?

Integra accesibilidad, contenido educativo, gamificacion, IA adaptativa y reportes familiares en una sola experiencia. No se enfoca en una sola discapacidad, sino en adaptar el flujo a distintos perfiles.

## 4. Como usa inteligencia artificial?

Usa Gemini AI para adaptar recomendaciones de contenido segun el perfil del nino, su progreso y el tipo de apoyo que necesita. La IA ayuda a personalizar, no reemplaza el criterio de docentes o familias.

## 5. Como se guardan los datos?

El MVP esta preparado para Firebase / Cloud Firestore. En modo real puede guardar usuarios, progreso, estrellas y reportes familiares. En modo prototipo puede funcionar con almacenamiento local del navegador.

## 6. Que parte del MVP ya funciona?

Funciona la interfaz principal, seleccion de perfil de accesibilidad, historia interactiva, lectura asistida, modo visual inclusivo, recompensas, panel familiar y demo publica en GitHub Pages.

## 7. Que falta para una version comercial?

Faltan autenticacion real, reglas completas de seguridad, pruebas con usuarios, mas contenido educativo, panel docente, analiticas, validacion pedagogica y acuerdos con instituciones.

## 8. Como se relaciona con los ODS?

Se relaciona con el ODS 4, educacion de calidad, porque busca acceso inclusivo al aprendizaje; y con el ODS 10, reduccion de desigualdades, porque reduce barreras para ninos con necesidades especificas.

## 9. Como se puede sostener economicamente?

Mediante licencias para escuelas y centros terapeuticos, convenios con fundaciones, una version familiar gratuita, planes premium y alianzas con programas de innovacion educativa o responsabilidad social.

## 10. Como validarian el impacto?

Con pruebas piloto midiendo participacion, finalizacion de actividades, tiempo de uso, comprension, satisfaccion familiar, facilidad de uso docente y progreso por perfil de accesibilidad.

## 11. Que riesgos tiene el proyecto?

Los principales riesgos son privacidad de datos infantiles, dependencia de servicios externos, calidad de respuestas de IA y necesidad de validacion pedagogica. Se mitigan con reglas de seguridad, supervision humana, pruebas y diseno responsable.

## 12. Cual es el siguiente paso despues del hackathon?

Ejecutar un piloto pequeno con familias o una institucion, mejorar la seguridad de Firebase, ampliar actividades, publicar una demo estable y preparar una version beta con usuarios reales.
